### 双指针

- **88. 合并两个有序数组**
  - **方法1：双指针 + 额外空间 + 拷贝**
  - **方法2：逆向双指针（从数组尾部开始遍历）**
  - **方法3：合并 + 排序**

- **27. 移除元素**
  - **方法1**：`a, b` 指针起始为 0  
    - `a` 指针遍历，`b` 指针赋值  
    - 如果 `arr[a] == val`：`b` 不动，`a++`  
    - 如果 `arr[a] != val`：先赋值 `arr[b] = arr[a]`，然后 `b++，a++`

- **26. 删除有序数组中的重复项**
  - **方法1**：`a, b` 指针起始位置为 1  
    - `a` 指针遍历，`b` 指针赋值  
    - 如果 `arr[b-1] == arr[a]`：`b` 不动，`a++`  
    - 如果 `arr[b-1] != arr[a]`：`b++，arr[b] = arr[a]，a++`

- **80. 删除有序数组中的重复项 II**
  - **方法1**：`a, b` 指针起始为 2  
    - `a` 指针遍历，`b` 指针赋值  
    - 如果 `arr[b-2] == arr[a]`：`a++`  
    - 如果 `arr[b-2] != arr[a]`：`arr[b] = arr[a]，a++，b++`  
    - **整体法**：可以将 `[b-2, b-1]` 看作一个整体

- **125. 验证回文串**
  - **方法1**：先转为小写，然后左右指针移动，相遇则是回文，否则不是

- **167. 两数之和 II - 输入有序数组**
  - **方法1**：数组已经有序，直接使用双指针

- **11. 盛最多水的容器**
  - **方法1**：双指针  
    - 宽度固定，两端指针向中间收缩  
    - 规则：  
      - 如果 `x < y`：移动较小的指针（`x++`）  
      - 如果 `x > y`：移动较小的指针（`y--`）  
      - 如果 `x == y`：`x++` 或 `y--`

- **15. 三数之和**
  - **方法1**：
    1. 排序
    2. 固定一个数，如果重复则跳过
    3. 对剩余部分使用双指针
    4. 左指针遇到重复值时跳过，右指针不用特别处理

- **977. 有序数组的平方**
  - **方法1（双指针 + 结果从后往前填）**  
    - 思路：  
      - 由于负负得正，负数平方可能更大  
      - 又因为数组有序，最大值一定在两端（最左或最右）  
    - 流程：
      - 设 `left = 0`, `right = len(nums) - 1`, `i` 为结果数组末尾下标  
      - 如果 `nums[left]^2 > nums[right]^2`：  
        - `res[i] = nums[left]^2`，`left++，i--`  
      - 如果 `nums[left]^2 < nums[right]^2`：  
        - `res[i] = nums[right]^2`，`right--，i--`  
      - 如果相等：  
        - 任选一侧，例如 `res[i] = nums[right]^2`，`right--，i--`  
      - 结束条件：`left > right`

- **总结**
  - *2025-10-03*：双指针通常考虑的是**最左端和最右端**。

### 贪心

- **121. 买卖股票的最佳时机**
  - **方法1：贪心**
    - 公式：`max(prices[j] - prices[i])` 且 `j > i`
    - 当天想卖出时，之前的买入一定选历史最低价

- **122. 买卖股票的最佳时机 II**
  - **方法1：贪心**
    - 只要第二天价格比第一天高，就在这两天之间买入并卖出（累计所有上升段）

- **55. 跳跃游戏**
  - **方法1：贪心**
    - 维护一个当前能跳到的最远位置 `maxReach`  
    - 当遍历下标 `i` 时：  
      - 如果 `i > maxReach`：说明无法到达当前位置，返回 false  
      - 不断更新 `maxReach = max(maxReach, i + nums[i])`
    - 如果 `maxReach >= len(nums) - 1`，则可以到达终点

- **45. 跳跃游戏 II**
  - **方法1：贪心**
    - 维护当前层（当前跳跃可覆盖的最远位置）和下一层的最远位置，每层更新一次跳跃次数  
    - “更新了多少次当前能跳到的最远位置，就代表最小跳跃次数”

- **134. 加油站**
  - **方法1：局部最优 + 全局最优**
    - 局部最优：当前剩余总量 < 0，则认为从当前起点不能走完，起点移到 i+1
    - 全局最优：如果总的 `(gas[i] - cost[i])` 之和 ≥ 0，则一定存在解  
  - 直观解释：
    - 首先判断总 `gas` 是否 ≥ 总 `cost`，否则无解  
    - 遍历中维护累计差值的最低点，解就是最低点的下一个位置
    - 从最低点后开始，后面“赚到的油”会补回之前的亏损

- **135. 分发糖果**
  - **方法1：两次遍历**
    1. 从左到右遍历，如果 `ratings[i] > ratings[i-1]`，则 `left[i] = left[i-1] + 1`
    2. 从右到左遍历，如果 `ratings[i] > ratings[i+1]`，则 `right[i] = right[i+1] + 1`
    3. 每个位置的糖果数为 `max(left[i], right[i])`，总和即为答案

- **92. 反转链表 II**
  - **方法1：三指针局部反转（头插法）**
    - `cur`：指向待反转区域第一个节点 `left`
    - `next`：永远指向 `cur` 的下一个节点（随着循环更新）
    - `pre`：永远指向 `left` 前一个节点（不变）
    - 循环步骤：
      ```text
      next = cur.next
      cur.next = next.next
      next.next = pre.next
      pre.next = next
      ```
    - 示例：`9→7→4→5→2→3→6, left=2, right=4`
      1. 将节点 5 插入到节点 2 前面
      2. 将节点 4 插入到节点 5 前面
      3. 将节点 3 插入到节点 4 前面
  - **方法2**：找到前一个位置，然后对 [left, right] 子链表整体反转，再重新连接

      ```go
      }
      ```

    ```go
      if root == nil {
        return 0
      }
      left := getNodeHeight(root.Left)
      right := getNodeHeight(root.Right)
      return max(left, right) + 1
    }
    ```

- **33. 搜索旋转排序数组**
  - **方法1**：二分查找 + 判断有序区间
    1. 若 `nums[mid] == target`：返回 `mid`
    2. 若 `nums[mid] >= nums[left]`：`mid` 在左侧有序区
       - 判断 `target` 是否在 `[left, mid-1]`：
         - 在：`right = mid - 1`
         - 否：`left = mid + 1`
    3. 否则 `mid` 在右侧有序区：
       - 判断 `target` 是否在 `[mid+1, right]`：
         - 在：`left = mid + 1`
         - 否：`right = mid - 1`
    - 直到 `left > right` 结束  
    - 提示：
      - 核心是利用“哪一侧有序”缩小查找区间  
      - `nums[mid] >= nums[left]` 时，`[left, mid]` 一定有序

- **153. 寻找旋转排序数组中的最小值**
  - **方法1**：二分 + 有序区间
    - 利用与 33 题相似思路，每次根据有序的一侧更新答案  
    - 在 `[left, mid]` 有序时，`minVal = min(minVal, nums[left])`

### 动态规划

#### 一维动态规划

- **70. 爬楼梯**
  - 状态：`dp[i]` 表示到达第 `i` 阶的方法总数  
  - 转移：`dp[i] = dp[i-1] + dp[i-2]`

- **198. 打家劫舍**
  - 状态：`dp[i]` 表示偷到第 `i` 家时的最大金额  
  - 转移：`dp[i] = max(dp[i-2] + nums[i], dp[i-1])`

- **300. 最长递增子序列**
  - 状态：`dp[i]` 表示以 `i` 结尾的最长递增子序列长度  
  - 转移：`dp[i] = max(dp[i], dp[j] + 1)`（对所有 `j < i` 且 `nums[j] < nums[i]`）

#### 背包问题

- **322. 零钱兑换**
  - 状态：`dp[i]` 表示凑成金额 `i` 所需的最少硬币数  
  - 转移：`dp[i] = min(dp[i], dp[i - coins[j]] + 1)`  
  - 初始化：`dp[0] = 0`，其他为 `amount + 1`（代表无穷大）  
  - 顺序问题（物品、背包遍历顺序）是个易混点

#### 多维动态规划

- **63. 不同路径 II**
  - 状态：`dp[i][j]` 表示到 `(i, j)` 的路径数  
  - 初始：如果没有障碍，`dp[i][0] = 1`, `dp[0][j] = 1`  
  - 若 `grid[i][j] == 1`（障碍）：`dp[i][j] = 0`  
  - 否则：`dp[i][j] = dp[i-1][j] + dp[i][j-1]`

- **64. 最小路径和**
  - 状态：`dp[i][j]` 表示到 `(i, j)` 的最小路径和  
  - 边界：
    - `i-1 < 0` 且 `j-1 >= 0`：`dp[i][j] = dp[i][j-1] + grid[i][j]`
    - `i-1 >= 0` 且 `j-1 < 0`：`dp[i][j] = dp[i-1][j] + grid[i][j]`
  - 一般情况：
    - `dp[i][j] = min(dp[i-1][j], dp[i][j-1]) + grid[i][j]`

- **120. 三角形最小路径和**
  - 状态：`dp[i][j]` 表示到三角形中 `(i, j)` 的最小路径和  
  - 结果为最后一行 `dp[n-1][j]` 的最小值

- **97. 交错字符串**
  - 状态：`dp[i][j]` 表示 `s1` 前 `i` 个 + `s2` 前 `j` 个能否交错组成 `s3` 前 `i+j` 个  
  - 转移：
    ```text
    dp[i][j] = (s1[i] == s3[i+j] && dp[i-1][j]) ||
               (s2[j] == s3[i+j] && dp[i][j-1])
    ```
  - 初始化：`dp[0][0] = true`

- **5. 最长回文子串**
  - 状态：`dp[i][j]` 表示子串 `s[i..j]` 是否为回文  
  - 转移：
    ```text
    dp[i][j] = (s[i] == s[j]) && (j - i < 3 || dp[i+1][j-1])
    ```
  - 答案是所有 `dp[i][j] == true` 中 `(j - i)` 最大的一段

### 大数计算

- **无符号大数相加**
  - 关键：当前位 `a`，当前位 `b`，进位 `carry`
  - 当前位：`(a + b + carry) % 10`
  - 新进位：`(a + b + carry) / 10`

- **无符号大数相减**
  - 关键：当前位 `a`，当前位 `b`，借位 `borrow`
  - 先保证“大数 - 小数”，最终根据是否交换决定符号  
  - 若 `a - b - borrow < 0`：
    - 当前位：`a - b - borrow + 10`，`borrow = 1`
  - 否则：
    - 当前位：`a - b - borrow`，`borrow = 0`
  - 最后去掉高位多余的 0

- **有符号大数加减**
  - 可以转化为“无符号大数加法/减法”

- **大数相乘**
  - 两层循环，结果存入数组，对应位不断累加，同时处理进位（可能 > 1）

- **大数相除**
  - 实现较复杂，面试中一般不要求手写完整大数除法

### 单调结构体

#### 单调队列

- **队列概念**：
  - 在数组中 `index = 0` 为队头，`index = len(queue)-1` 为队尾  
  - 特性：先进先出（FIFO）

- **单调队列**：
  - 可以理解为双端队列，为维护单调性，会从队尾出队  
  - 常用于“滑动窗口最大/最小值”

- **单调递增队列（队列元素从队头到队尾递增）**
  - 维护过程：
    - 插入新元素 `x` 时，从队尾开始**弹出所有比 `x` 大的元素**，直到队尾元素 ≤ `x`，再将 `x` 入队
  - 作用：**快速获取某个区间内的最小值**（队首即为当前区间的最小值）

- **单调递减队列（队列元素从队头到队尾递减）**
  - 维护过程：
    - 插入新元素 `x` 时，从队尾开始**弹出所有比 `x` 小的元素**，直到队尾元素 ≥ `x`，再将 `x` 入队
  - 作用：**快速获取某个区间内的最大值**

- **239. 滑动窗口最大值**
  - **方法1：单调递减队列**
    1. 初始化先处理下标 `0 ~ k-1`
    2. 之后每一步：
       - 先主动出队（窗口外的元素）
       - 再根据单调性入队
       - 队首即为当前窗口最大值

- **42. 接雨水**
  - 单调递减队列或前后缀最高值预处理  
  - 核心：
    - 对每个位置 `i`：
      - 水量 = `max(0, min(左边最高, 右边最高) - 当前高度)`

- **402. 移掉 K 位数字**
  - **方法1：贪心 + 单调递增“栈/队列”**
    - 维护一个尽量单调递增的结构，每当当前数字小于栈顶且 `k > 0` 时弹出栈顶  
    - 最终去掉多余的末尾位，再去掉前导零

#### 单调栈

- **栈概念**：
  - 数组中 `index = 0` 为栈底，`index = len(stack)-1` 为栈顶  
  - 特性：先进后出（LIFO）

- **单调栈**：
  - 为保证从栈底到栈顶单调递增或单调递减

- **单调递增栈**
  - 保证栈底到栈顶是递增（遇到小值时弹栈）
  - 应用：寻找**右边或左边第一个比当前小的元素**

- **单调递减栈**
  - 保证栈底到栈顶是递减（遇到大值时弹栈）
  - 应用：寻找**右边或左边第一个比当前大的元素**

- **456. 132 模式**
  - **方法1**：暴力三层循环（TLE 风险）
